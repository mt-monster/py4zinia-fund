# 投资策略优化方案

## 一、问题分析

### 现有策略的痛点

| 问题 | 说明 |
|-----|------|
| **过于细分** | 11个策略，阈值划分过于精细（0.3%、0.5%、1.0%等）|
| **操作同质化** | 6个策略都是"持有观望"，用户无法感知差异 |
| **覆盖不全** | 0-0.3%、0.01%等狭窄区间容易出现"未知状态"|
| **认知负担** | 策略名称过多，用户难以理解和记忆 |

### 现有策略操作分布

```
强势突破 → 持有
持续上涨 → 持有
温和上涨 → 持有
上涨放缓 → 持有  ← 4个策略相同操作！
反转上涨 → 持有
转势休整 → 弱卖
反转下跌 → 卖出
绝对企稳 → 买入
首次大跌 → 强买
持续下跌 → 买入  ← 2个策略相同操作！
跌速放缓 → 买入
```

## 二、优化方案

### 核心原则
1. **精简策略数量**：从11个简化为6个
2. **明确操作差异**：每个策略有独特的操作建议
3. **扩大覆盖范围**：减少细分阈值，避免盲区
4. **用户易理解**：策略名称直观，操作清晰

### 新旧策略对比

| 新策略 | 操作 | 覆盖原策略 | 简化说明 |
|-------|------|-----------|---------|
| 大涨止盈 | 弱卖(5%) | 强势突破+持续上涨 | 合并连续大涨情况 |
| 温和上涨 | 持有 | 温和上涨+上涨放缓 | 合并所有上涨情况 |
| 反转上涨 | 持有 | 反转上涨 | 保留，趋势转变信号 |
| 反转下跌 | 卖出(8%) | 反转下跌+转势休整 | 合并由涨转跌情况 |
| 小跌买入 | 买入(1.5x) | 绝对企稳+持续下跌+跌速放缓 | 合并正常下跌情况 |
| 大跌抄底 | 强买(2.5x) | 首次大跌 | 保留，极端情况 |

### 阈值简化

| 维度 | 优化前 | 优化后 |
|-----|--------|--------|
| 上涨划分 | 0.3%、0.5%、1.0%三档 | 0.5%一档 |
| 下跌划分 | 0.5%、1.0%、2.0%三档 | 1.0%一档 |
| 持平区间 | 0-0.01%（过窄） | 并入相邻区间 |

## 三、策略决策矩阵

```
                    昨日表现
              上涨      下跌      持平
         ┌─────────┬─────────┬─────────┐
  上涨   │ 大涨止盈 │ 反转上涨 │ 温和上涨 │
         │ (弱卖)  │ (持有)  │ (持有)  │
今日     ├─────────┼─────────┼─────────┤
  下跌   │ 反转下跌 │ 小跌买入 │ 小跌买入 │
         │ (卖出)  │ (买入)  │ (买入)  │
         ├─────────┼─────────┼─────────┤
  大跌   │ 大跌抄底 │ 大跌抄底 │ 大跌抄底 │
         │ (强买)  │ (强买)  │ (强买)  │
         └─────────┴─────────┴─────────┘
```

## 四、优势对比

| 维度 | 优化前 | 优化后 |
|-----|--------|--------|
| 策略数量 | 11个 | 6个（减少45%）|
| 阈值档位 | 7档 | 3档 |
| 操作类型 | 6种 | 4种（更聚焦）|
| 覆盖盲区 | 较多 | 极少 |
| 用户理解 | 困难 | 容易 |

## 五、实施建议

### 1. 渐进式迁移
```
Phase 1: 同时支持新旧策略配置
Phase 2: 新数据使用新策略，历史数据兼容
Phase 3: 完全切换到新策略
```

### 2. A/B测试
- 对比新旧策略的用户理解和操作准确性
- 监控"未知状态"出现频率

### 3. 用户反馈
- 收集用户对策略名称和操作建议的反馈
- 根据实际效果微调阈值

## 六、预期效果

1. **减少"未知状态"**：覆盖更全面，预计减少80%
2. **降低认知负担**：策略更易理解，用户决策更快
3. **操作更聚焦**：买入/卖出/持有三种主要操作更清晰
4. **维护更简单**：策略数量减半，配置更易管理
